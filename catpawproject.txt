	this will damage processor if mismated
	PSU is Black
	Ethercat is Green

	1	Us		Brown
	2	Up		White
	3	Gnds		Blue
	4	Gndp		Black
	Shield	N/A		not connected

	1	Tx+

			(Not GND)

parsing the i/o

the catpaw is the sensor.
the beckhoff is the device.

480v is the input to the catpaw which is the data
	goes to divider and buffer to derive the data
	the output of which goes to the beckhoff analog input +/-
		analog input 1: pin 2, 4

	typically 
		with an external supply connected to beckhoff m8 power jack
		power would be supplied to the sensor from pin 1, 3 on that m12
		from the beckhoff m8 Up bus
		m12 pin 1,3 would power the catpaw's active buffer
		
in our case:
480v from catpaw input supplies power to the Rak 24Vdc PSU
	which in turn powers the active buffer
	this would also backfeed 24vDC through the m12, pin1,3 onto the Up bus in the beckhoff

a switch will disconnect 480v from the rak psu
	unpowering the buffer from the 480v
	480v will still make it to the divider
	
	if an external psu was connected to the beckhoff via m8 power
	power would still reach the buffer via m12 pin1,3

we want to use the Rak psu to also provide beckhoff m8 power input
	catpaw will have an m8 power output
	to be connected to the bottom m8 beckhoff power input
	we would bridge Us and Up before out power output connector
so
	catpaw m8
	1	pwrout+	Us
	2	pwrout+	Up
	3	pwrout-	GNDs
	4	pwrout-	Gndp
this allows the device to be powered from the 480v catpaw input (through rakPSU)
	rakpsu supplies Up device voltage
			Us sensor voltage
Us sensor voltage comes through m12 to pwrout+/-
	they should be at the same potential (besides cable voltage drop)
